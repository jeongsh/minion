# 페이지별 콘텐츠 완성도 검수

검수 기준: 코드 버그가 아니라 페이지 목적, 핵심 정보 우선순위, 정보량, 대표 행동, 콘텐츠 정리 상태를 기준으로 봤다.

검수 근거:
- 메인: `app/page.tsx`
- 일정: `app/schedule/page.tsx`
- 순위: `app/standings/page.tsx`
- 팀 목록/상세: `app/teams/page.tsx`, `app/teams/[teamSlug]/page.tsx`
- 선수 목록/상세: `app/players/page.tsx`, `app/players/[playerSlug]/page.tsx`
- 경기/세트 상세: `app/matches/[matchId]/page.tsx`, `app/matches/[matchId]/sets/[setId]/page.tsx`
- 팬사이트: `app/fan/[teamSlug]/*`
- 통계/커뮤니티: `app/stats/*`, `app/community/*`

브라우저 확인:
- 데스크톱 대표 라우트: `/`, `/schedule`, `/standings`, `/teams`, `/players`, `/stats`, `/community`, `/fan/t1`
- 모바일 확인: 390 x 844 기준
- 주요 관찰: 메인 모바일 스크롤 약 4718px, 선수 목록 약 9313px, 팬 홈 약 4705px. 순위표와 일부 통계표는 내부 테이블이 42rem 이상이라 모바일에서 가로 탐색 피로가 큼.

## 1. 메인 페이지

1. 현재 페이지의 목적:
   - 경기, 순위, 최근 폼, 영상, 팬사이트, 평점 참여, 광고를 한 화면에 모아 보여준다.

2. 더 명확하게 바꾼 페이지 목적:
   - 오늘 또는 이번 주에 사용자가 봐야 할 LCK 핵심만 빠르게 알려준다.

3. 이 페이지가 답해야 하는 핵심 질문:
   - 오늘 무엇을 보면 되는가?

4. 사용자가 가장 먼저 봐야 할 정보:
   - 1순위: 다가오는 경기 1-2개
   - 2순위: 현재 리그 구도 요약, 예를 들어 순위 TOP5
   - 3순위: 지금 참여 가능한 평점/승부예측 또는 볼 만한 영상 1개

5. 현재 너무 많이 보이는 정보:
   - 월간 캘린더 전체
   - 순위표와 최근 폼 표의 동시 노출
   - 추천 영상 4개
   - 팀별 팬사이트 로고 10개
   - 광고 2개

6. 핵심을 방해하는 정보:
   - 히어로 슬라이더가 실제 핵심 경기보다 먼저 시선을 가져간다.
   - 광고가 순위/최근 폼과 같은 시각 무게로 놓인다.
   - 캐릭터 이미지 placeholder는 콘텐츠가 아니라 임시 장식처럼 보인다.

7. 반드시 남겨야 할 콘텐츠:
   - 다가오는 경기
   - 경기 상세 CTA
   - 순위 TOP5
   - 현재 열려 있는 팬 참여 CTA

8. 접기/더보기 처리해야 할 콘텐츠:
   - 월간 캘린더
   - 최근 폼 전체
   - 추천 영상 목록

9. 상세 페이지로 보내야 할 콘텐츠:
   - 전체 일정: `/schedule`
   - 전체 순위/상세 지표: `/standings`, `/stats`
   - 전체 영상: 팬 영상 페이지 또는 별도 영상 영역

10. 삭제해도 되는 콘텐츠:
    - 캐릭터 이미지 placeholder
    - 중복 광고 슬롯 중 하나

11. 대표 행동 1개:
    - 경기 상세 보기

12. 보조 행동 1개:
    - 평점 또는 승부예측 참여

13. 사용자가 헷갈릴 수 있는 부분:
    - 메인이 일정 페이지인지, 투표 페이지인지, 영상 페이지인지 목적이 섞여 보인다.

14. 모바일에서 문제가 될 수 있는 부분:
    - 첫 화면 이후 스크롤이 길고 섹션 수가 많다.
    - 캘린더와 목록이 연속 노출되어 피로하다.

15. 로딩/빈 상태/에러 상태에서 필요한 처리:
    - 경기 없음: "오늘 예정된 경기가 없습니다. 이번 주 일정 보기"
    - 영상 없음: "추천 영상이 아직 없습니다. 팀별 공식 채널 보기"
    - 순위 없음: "현재 시즌 순위가 아직 집계되지 않았습니다."

16. 개선 방향:
    - 첫 화면을 경기 중심으로 재구성한다.
    - 캘린더는 요약형으로 줄이고 전체 일정으로 이동시킨다.
    - 광고는 핵심 경기/참여 CTA 사이에 두지 않는다.

# 전체 사이트 기준 정리

## 1. 전체 평가

잘 구성된 부분:
- 일정 페이지는 핵심 질문이 가장 명확하다.
- 경기 상세는 탭 구조 덕분에 프리뷰, 데이터, 평점, 영상의 경계가 비교적 살아 있다.
- 팬 영상 페이지는 "지금 볼 영상"이라는 목적이 분명하다.

산발적으로 느껴지는 부분:
- 메인, 선수 상세, 팬 홈.
- 세 페이지 모두 섹션이 많고 CTA가 여러 개라 첫 화면에서 무엇을 해야 하는지 약하다.

사용자가 핵심을 놓칠 가능성이 큰 부분:
- 메인의 광고/영상/팀바로가기 혼재
- 선수 상세의 세트별 기록 과다 노출
- 팬 홈의 인스타그램 긴 본문
- 순위표의 세부 지표 컬럼

## 2. 가장 문제가 큰 페이지 TOP 5

1. 팬 커뮤니티 하위 placeholder
   - 빈 페이지인데 탭에 노출된다.

2. 선수 상세
   - 선수 프로필, 통계 대시보드, 경기 로그, 챔피언 분석, 팀원 탐색을 모두 수행한다.

3. 선수 목록
   - 모바일 스크롤이 9000px 이상으로 길다.

4. 팬사이트 홈
   - 팀 홈인데 SNS/인스타/영상이 홈을 장악한다.

5. 메인
   - "오늘 볼 것"보다 여러 위젯의 병렬 배치가 강하다.

## 3. 정보량을 줄여야 하는 영역 TOP 5

1. 선수 상세 최근 경기 기록
   - 기본 3경기 요약만 남기고 세트별 아이템/룬은 접는다.

2. 팬 홈 인스타/SNS
   - 최신 2-3개 카드만 기본 노출한다.

3. 순위표
   - KDA, 킬, 데스, 어시스트는 `/stats/teams`로 이동한다.

4. 메인 캘린더
   - 이번 주/다음 경기만 기본 노출하고 월간 달력은 일정 페이지로 보낸다.

5. 선수 목록
   - 팀별 아코디언 또는 팀 필터를 기본으로 둔다.

## 4. 메인 페이지에 반드시 남겨야 할 콘텐츠

1. 오늘/다가오는 경기 1-2개와 경기 상세 CTA
2. 현재 리그 구도: 순위 TOP5 또는 한 줄 요약
3. 참여 CTA: 평점/승부예측 중 현재 열려 있는 것 1개
4. 최신 볼거리: 영상 1개 또는 팀 소식 2개
5. 팀 팬사이트 진입: 하단 compact 로고 그리드

## 5. 메인 페이지에서 빼거나 아래로 내려야 할 콘텐츠

- 월간 캘린더 전체
  - 이유: 메인 목적보다 일정 페이지 목적에 가깝다.

- 팀 최근 폼 전체 표
  - 이유: 순위 보조 정보로 충분하다.

- 추천 영상 4개
  - 이유: 메인에서는 대표 영상 1개면 충분하다.

- 팀 바로가기 10개
  - 이유: 하단 진입 영역이면 충분하다.

- 광고 2개
  - 이유: 핵심 콘텐츠 사이에 들어가면 흐름을 끊는다.

## 6. 상세 페이지로 이동시켜야 할 콘텐츠

- 순위 세부 지표: `/stats/teams`
- 선수 목록의 실명/프로필 상세: `/players/[playerSlug]`

- 팬 홈의 전체 영상: `/fan/[teamSlug]/videos`
- 팬 홈의 전체 선수: `/fan/[teamSlug]/players`
- 팀 상세의 전체 경기 이력: `/fan/[teamSlug]/matches` 또는 일정의 팀 필터 화면

## 7. 접기/더보기 처리해야 할 콘텐츠

- 메인 경기 2개

- 선수 상세 챔피언 표, 경력, 수상, 팀원
- 세트 상세 전체 선수 스탯, 타임라인
- 팀 상세 전체 경기, 전체 수상

## 8. 삭제 후보 콘텐츠

- 메인 캐릭터 이미지 placeholder
  - 콘텐츠 의미가 없다.

- 팀 목록 감독/코치 요약
  - 팀 선택 단계의 핵심이 아니다.

- 팬 정보 페이지의 표 형식
  - 공식 링크는 버튼 카드가 더 적합하다.

- 미구현 팬 커뮤니티 하위 탭
  - 빈 기능으로 사용자를 보낸다.

- VOD 없는 경기의 영상 탭
  - 빈 탭을 만들 필요가 없다.

- "경기장 미정" 반복 노출
  - 정보 가치가 낮고 노이즈가 된다.

## 9. 모바일에서 가장 먼저 고쳐야 할 문제

터치:
- 탭/필터가 많은 페이지는 대표 CTA를 먼저 배치한다.

스크롤:
- 메인, 선수 목록, 팬 홈을 절반 이하 길이로 줄인다.

정보 밀도:
- DataTable은 모바일 요약 카드로 대체하고 세부 컬럼은 접는다.

광고 배치:
- 경기 카드 사이, 순위 표 위, 팬 홈 다음 경기 직후에는 넣지 않는다.

## 10. 광고 영역 검토

광고가 콘텐츠 흐름을 방해하는지:
- 메인 광고는 순위/폼/영상 흐름 사이에 들어가 핵심 콘텐츠처럼 보인다.

광고를 유지하되 덜 거슬리게 배치할 방법:
- 첫 화면 밖에 둔다.
- 섹션 끝에 낮은 높이 배너로 둔다.
- 광고 슬롯 스타일을 콘텐츠 카드와 명확히 구분한다.

광고 때문에 핵심 콘텐츠가 밀리는 부분:
- 메인의 순위/최근 폼과 영상 사이
- 하단 팀 바로가기 전후

## 11. 데이터 없음/에러 상태 검토

유튜브 썸네일 없음:
- 문구: "썸네일을 불러오지 못했습니다. YouTube에서 보기"
- UI: 고정 비율 검은 fallback, 플레이 아이콘 유지

영상 없음:
- 문구: "아직 동기화된 영상이 없습니다. 공식 YouTube 채널 보기"
- UI: 빈 카드 1개와 외부 채널 CTA

뉴스 없음:
- 문구: "최근 등록된 팀 소식이 없습니다. 공식 SNS 보기"
- UI: 빈 상태와 공식 링크

경기 없음:
- 문구: "선택한 기간에 경기가 없습니다. 다음 달 일정 보기"
- UI: 기간 이동 버튼

순위 데이터 없음:
- 문구: "이 구간은 아직 순위가 집계되지 않았습니다. 전체 시즌 보기"
- UI: 전체 시즌 CTA

API 실패:
- 문구: "데이터를 불러오지 못했습니다. 잠시 후 다시 시도해주세요"
- UI: 재시도 버튼

이미지 로딩 실패:
- 문구: 별도 문구보다 이니셜 fallback 권장
- UI: 팀/선수 이니셜, 고정 크기 유지
